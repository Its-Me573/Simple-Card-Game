# Personal Card Game Project – Early OOP Practice

## Purpose
This project was created to practice object-oriented programming in C++.  
It was an exercise in class design, object ownership, and implementing basic card game logic.

## Features Implemented
- `DeckOfCards` class with card shuffling algorithm  
- `playingUser` class for both player and house  
- `Game` class that owns the deck and user objects  
- Basic functions for dealing cards and printing the screen  

## Lessons Learned
- The importance of encapsulation (private vs public members)  
- How to structure classes so objects can communicate effectively  
- Avoiding unnecessary parameters when member variables exist  
- How to organize a controller (`Game`) separate from `main`  

## Status
- Not a fully functional card game  
- Some functions remain incomplete or redundant  
- Project archived to move on to more advanced work  

## Notes
This project served as a learning milestone and stepping stone for more advanced programming assignments.  
It is intentionally left in its original state to preserve the learning experience.
